Workshop Week 10

In this week's workshop, you will use the desktop version of the ontology editor Protégé and explore how an existing OWL ontology has been built with the help of this tool. Afterwards, you will build a simple family ontology starting from a specification in (controlled) natural language.

1. Exploring Protégé and the Pizza Ontology

The Pizza ontology is an ontology that has been developed for teaching purposes at the University of Manchester. Please download the pizza.owl ontology and do the following things:

  1. Start up the ontology editor Protégé that is installed on the lab machines. Once you see the interface (below), go to File > Open ... and load the pizza.owl ontology into the editor (alternatively, you can fetch the ontology directly from http://protege.stanford.edu/ontologies/pizza/pizza.owl via File > Open From URL ...).



  2. Select the menu item Entities > Classes, expand the class hierarchy in the class hierarchy pannel, and explore how this hierarchy has been structured.

  3. In this class hierarchy, you will find the class CheeseyPizza among other classes organised under Pizza, select CheeseyPizza and check the description of CheeseyPizza in the description pannel.

  4. Check the descriptions of a few other classes that occur in the class hierarchy and explore how you can change the description of a class in the description pannel using the class expression editor (you have to click on edit icon first in order to edit something).

  5. Now select the menu item Object Properties, expand the object property hierarchy, and explore how this hierarchy has been structured.

  6. Have a look at the characteristics of these object properties in the characteristics pannel and at their descriptions in the description pannel. You can find some background information about the property characteristics in the OWL 2 Primer.

  7. Now select the menu item Individuals, and check how the individulas (for example, France) are described.

  8. Now go back to the class hierarchy, select again CheesyPizza in the class hierarchy, have a look at the description pannel, then go to the menu item Reasoner, select HermiT, and then Start Reasoner, check the desription pannel again and find out what has changed. Do the same thing for a number of other classes.

  9. Now, select the menu item Windows > Tabs > SPAQRL Query, and execute the example query there. What's going on here? Try to come up with your own SPARQL queries and execute them over the Pizza ontology.

  10. Finally, select the menu item File > Save as ... and save the Pizza ontology in Turtle notation, open the resulting file with a text editor and have a quick look at it.

2. Building an OWL Ontology from a Specification in Natural Language

Now you should be ready to build your own ontology. The specification below speaks about a family and the relations between family members. This specfication is written in a (controlled) natural language and it should be relatively straightforward to modell this knowledge in Protégé, if you stick to the following conventions:

  1. modell proper names such as Cordula as individuals;

  2. modell nouns such as gender as classes;

  3. modell expressions such as has ... as a parent as roles (for example, hasParent);

  4. modell expressions such as has a person as a parent as a role followed by an existenital restriction (for example, (hasParent some Person));

  5. modell expressions such as has the value femaleGender as gender as a role, followed by the keyword value, followed by the corresponding individual (for example, (hasGender value femaleGender));

  6. modell the expression is defined as as equivalent to.

Specification in (controlled) natural language:

   Cordula has Rona as a sibling.
   Cordula has Erwin as sibling.
   Cordula has Mia as a parent.

   Every gender is a thing.
   Every person is a thing.

   Every daughter is a subclass of a person.
   Every daughter is defined as a person who has a person as a parent and 
                                         who has the value femaleGender as gender.

   Every father is a subclass of a person.
   Every father is defined as a person who has a person as a child and 
                                       who has the value maleGender as gender.

   Every mother is a subclass of a person.
   Every mother is defined as a person who has a person as a child and
                                       who has the value femaleGender as gender.

   Every offspring is a subclass of a person.
   Every offspring is defined as a person who has a person as a parent.

   Every parent is a subclass of a person.
   Every parent is defined as a person who has a person as a child.

   Every son is a subclass of a person.
   Every son is defined as a person who has a person as a parent and 
                                    who as the value maleGender as gender.


   Cordula is an individual.
   Cordula has Christa as a sibling.
   Cordula has Rona as a sibling.
   Cordula has Erwin as sibling.
   Cordula has Mia as a parent.


   Robert is an individual.

   Dave is an individual.
   Dave has Robert as a child.

   Mia is an individual.

   Jim is an individual.
   Jim has Peter as a sibling.
   Jim has Rona as a parent.

   Rona is an individual.
   Rona has Christa as a sibling.
   Rona has Robert as a parent.
   Rona has Mia as a parent.
   Rona has femaleGender as gender.
   Rona has Rona Smith as full name.

   Erwin is an individual.
   Erwin has Rona as a sibling.
   Erwin has Christa as a sibling.
   Erwin has Mia as a parent.
   Erwin has Robert as a parent.

   Christa is an individual.
   Christa has Rona as a sibling.
   Christa has Robert as a parent.
   Christa has Mia as a parent.

   Peter is an individual.
   Peter has maleGender as gender.
